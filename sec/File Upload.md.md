# 漏洞名称

文件上传漏洞（File Upload）

---

## 原理

应用提供文件上传功能时，未对上传文件进行严格校验，导致攻击者能够上传非预期文件。

如果上传后的文件可以被访问，并且服务器会对其进行解析执行，则可能造成严重安全问题。

本质原因：

- 服务端信任用户提交的文件
- 文件类型校验不严格
- 上传目录权限配置不当
- 上传后文件可被访问或执行

---

## 常见场景

- 用户头像上传
- 图片上传
- 附件上传
- 工单系统
- 富文本编辑器
- 企业认证资料上传
- 文件管理系统
- CMS后台
- Excel导入
- PDF上传

---

## 常见特征

- 存在文件上传功能
- 请求为 multipart/form-data
- 请求包中存在 filename 参数
- 请求包中存在 Content-Type 字段
- 上传成功后返回文件路径或文件ID
- 上传后的文件能够被访问

---

## 验证方法

### Payload

正常上传：

```text
test.jpg
```

重点观察：

- 修改文件名
- 修改文件后缀
- 修改 Content-Type
- 修改文件内容

观察系统是否仍然允许上传。

### 响应特征

上传成功：

```json
{
  "success": true
}
```

返回路径：

```json
{
  "url": "/upload/test.jpg"
}
```

返回文件ID：

```json
{
  "id": "12345"
}
```

需要继续寻找文件访问位置。

访问上传文件后：

情况一：

```text
直接下载
```

说明未执行。

情况二：

```text
正常显示图片
```

说明按静态资源处理。

情况三：

```text
服务器解析执行
```

说明存在高风险。

---

## 影响

低风险：

- 任意文件上传
- 非授权文件存储

中风险：

- 上传恶意脚本
- 文件覆盖

高风险：

- 远程代码执行（RCE）
- 获取服务器权限
- 获取数据库权限
- 横向移动
- 服务器失陷

---

## 修复方案

### 文件类型校验

仅允许上传：

```text
jpg
jpeg
png
gif
pdf
```

### MIME校验

校验真实文件类型。

不要仅依赖：

```http
Content-Type
```

### 文件内容校验

检查文件头特征。

例如：

```text
JPEG
PNG
GIF
```

### 文件重命名

上传后随机生成文件名：

```text
a8f3d92f.jpg
```

避免用户控制文件名。

### 上传目录禁止执行

上传目录仅用于存储文件：

```text
/upload/
```

禁止脚本执行。

### 最小权限原则

上传目录仅授予必要权限。
